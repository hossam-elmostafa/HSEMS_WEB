## 5. Data consistency

- **Existing rows:** No backfill required. Triggers only affect future INSERT/UPDATE/DELETE.
- **Whitelist seed:** `HSE_CRENTRY_CRSTT_transition_guard` includes `('', '01')` and `('', '05')` for new + observation-generated CARs. Extend for any other policy-driven initial status.
- **Tracing migration:** Switching `ENTRY_COMPLETED` handlers from raw `HSE_CRENTRY_TRC` INSERT to `ChangeEntityStatus` affects **new** events only; historical TRC rows are unchanged.
- **`rejectCARTXN` 5th argument:** Web passes reject-reason row key (from `fullRecord`) when available, else `PrmKy`. Verify against live SP definition and adjust if the SP expects a different correlation key.
- **`ChangeEntityStatus` arg 7:** Documented as "prior `CRENTRY_CRSTT` fetched from DB before the action" (status token). Verify against `sys.sql_modules` for `ChangeEntityStatus` in your DB.

## 7. Testing strategy

### Manual / integration

- **ENTRY_COMPLETED** (→`05`): watch SQL trace — `ChangeEntityStatus` SP call must appear, no raw `HSE_CRENTRY_TRC` INSERT from app.
- **ENTRY_COMPLETED** (→`16`): same; also verify mass-update of corrections/corrective-actions flags.
- **Actions Review → accept** (→`19`): `ChangeEntityStatus` + target date rollup SQL.
- **CAR Review → accept** (→`10`): `ChangeEntityStatus` + policy row-count check.
- **APPROVE_CAR** (→`15`): `ChangeEntityStatus` + dept/project copy from `HSE_CRAPPRVLINF`.
- **REJECT_CAR** on follow-up (→`15`): blocking `confirm` shown; Cancel aborts reject screen.
- **CAR Entry**: change `CRENTRY_CRDT` across calendar years → `CRENTRY_CRYR` and `CRENTRY_CRN` refresh.
- **CAR Review**: change `CRENTRY_CRSRC` → `CRENTRY_CRSRLN` refresh.
- **Concurrent CAR creation**: two sessions same year → unique `(CRENTRY_CRYR, CRENTRY_CRN)`.
- **Observation → CAR**: confirm `CopyImages` runs; no duplicate-key error.
- **Auto-generated CAR**: DELETE blocked (UI + trigger).
- **Close CAR**: corrections/CAs with mixed-case `Y`/`y` still validate (collation-safe SQL).
- **Actions Review `ShowScreen`**: `CRENTRY_RVWDT` populated from server date (check Network tab → SQL contains `CONVERT(varchar(10), GETDATE(), 23)`).
- **Follow-up Corrections tab reposition**: `ACTNSENTRY_CRR_ACTFLLDT` auto-set from server date.
- **Temp table / pooling**: New CAR Entry save from web (temp absent) → trigger must not raise `Invalid object name '##TEMP_HSE_TABLE'` (requires DBA to apply `HSE_CAR_trigger_hardening_apply_RQ_HSE_12_4_26_00_40.sql` to real triggers).
